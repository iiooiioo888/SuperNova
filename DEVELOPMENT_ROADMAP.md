# SuperHub 开发路线图

本文档将项目开发拆分为可执行的阶段和任务，便于在 GitHub 中追踪进度。

## 📋 总体策略

采用**渐进式开发**策略，优先完成核心功能，再逐步扩展：

1. **Phase 0**: 基础设施准备（已完成 ✅）
2. **Phase 1**: 核心采集能力（B 站完整实现）
3. **Phase 2**: 任务调度与容错
4. **Phase 3**: 多平台扩展
5. **Phase 4**: 前端与管理界面
6. **Phase 5**: 高级功能与优化

---

- [x] 项目结构搭建

## Phase 1: 核心采集能力（B 站完整实现）

**目标**: 完成 Bilibili 适配器的全功能实现，验证架构设计

### 任务列表

- [ ] **#1 - 完善 B 站适配器标准接口**
  - 实现 `fetch_user_profile()`
  - 实现 `fetch_posts()` 支持分页
  - 实现 `fetch_comments()`
  - 实现 `download_media()`
  - 实现 `search()`
  - 单元测试覆盖率 > 80%

- [ ] **#2 - B 站特有功能（Capabilities）**
  - 直播弹幕采集
  - 收藏夹采集
  - UP 主视频列表批量采集

- [ ] **#3 - MongoDB 原始存储层**
  - 实现 `raw_store.py`
  - 设计原始数据文档结构
  - 写入性能优化（批量插入）

- [ ] **#4 - Elasticsearch 索引同步**
  - 设计 ES 索引映射
  - 实现异步同步任务
  - 搜索 API 实现

- [ ] **#5 - 对象存储集成**
  - MinIO/OSS 配置
  - 媒体文件下载与存储
  - CDN 集成方案

**验收标准**: 
- 可以成功采集 B 站用户资料、帖子、评论
- 数据正确存入三层存储
- 媒体文件可下载并访问

## Phase 2: 任务调度与容错

**目标**: 建立可靠的任务调度和容错机制

### 任务列表

- [ ] **#6 - Celery 任务系统完善**
  - 通用任务定义（采集用户/帖子/评论）
  - 任务优先级队列
  - 任务超时控制

- [ ] **#7 - 差异化重试策略**
  - 网络超时：指数退避
  - 频率限制：读取 Retry-After
  - 账号封禁：切换账号
  - 内容删除：跳过记录

- [ ] **#8 - 限流器实现**
  - 平台级限流配置
  - 账号级限流
  - IP 级限流
  - Token Bucket 算法

- [ ] **#9 - 代理池服务**
  - 代理获取（免费/付费 API）
  - 代理健康检查
  - 代理评分系统
  - 自动切换策略

- [ ] **#10 - 告警服务**
  - 熔断触发告警
  - 连续失败告警
  - 数据异常告警
  - Webhook/邮件/钉钉通知

**验收标准**:
- 任务失败自动重试符合策略
- 熔断器正常工作
- 限流器防止请求过载
- 异常情况及时告警

## 贡献指南

1. 从 GitHub Issues 领取任务
2. 创建 feature 分支：`git checkout -b feature/issue-{编号}-{简短描述}`
3. 开发完成后提交 PR
4. 通过 Code Review 后合并到 develop 分支
